# Project Overview
This project is a responsive frontend dashboard built for the Hintro Frontend Developer Internship assignment.

The dashboard closely follows the provided Figma design and integrates with the official Hintro Mock API to dynamically handle:

- New User / Empty State (`u1`)
- Active User / Populated State (`u2`)

---

# Features

## Core Features
- Responsive dashboard UI
- Figma-based layout implementation
- Dynamic API integration
- User state switching (`u1` / `u2`)
- Dashboard statistics
- Profile data rendering
- Subscription details
- Call session statistics
- Empty state handling
- Loading states
- Error handling
- Global theme styling (no hardcoded design inconsistency)
